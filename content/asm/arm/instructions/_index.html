ARM instruction set

The ARM instruction set consists of 32 bit word-aligned words based on the following format:

31 30 29 28 27 26 25 24 23 … 6 5 4 3 2 1 0
cond op1 op2
cond op1 op2 Instruction classes
not 1111 00x x Data processing & miscellaneous
010 x Load/Store word and unsigned byte
011 0 Load/Store word and unsigned byte
1 Media instructions
10x x Branch & block data transfer
11x x Coprocessor instructions & Supervisor Call
1111 xxx x Unconditional instructions
Advanced SIMD instructions
Floating Point instructions

The cond field when it is not set to 1111 represents one of the following conditions:

cond Mnemonic Meaning Flags
Integer Floating Point
0000 EQ Equal Z = 1
0001 NE Not Equal Not Equal, or unordered Z = 0
0010 CS Carry Set Greater than, equal, or unordered C = 1
0011 CC Carry Clear Less than C = 0
0100 MI Minus, Negative Less than N = 1
0101 PL Plus, Positive Greater than, equal, or unordered N = 0
0110 VS Overflow Unordered V = 1
0111 VC No overflow Not unordered V = 0
1000 HI Unsigned higher Greater than, or unordered C = 1 and z = 0
1001 LS Unsigned lower or same Less than or equal C = 0 or z = 1
1010 GE Signed greater than or equal Greater than or equal N = V
1011 LT Signed less than Less than or unordered N != V
1100 GT Signed greater than Greater than Z = 0 and N = V
1101 LE Signed less than or equal Less than, equal, or unordered Z = 1 or N != V
1110 None or AL Always unconditional N/A
1111 See main table above, used for instructions that can only be used unconditionally.

In the above table, unordered means that at least one operand is NaN.

Also, assemblers allow the following synonyms:

Mnemonic Synonym
HS Unsigned higher or same is a synonym for CS
LO Unsigned lower is a synonym for CC
AL An optional mnemonic for always, except for IT instructions
